Rationale

Modernization

We took the old design and modernized to follow current trends and styling using bootstrap. It's important to keep up with trends as this will convey to students the learning outcomes of the course. Color is also more streamlined and adapted to BCIT colors.

CSS reset

This allows the website to be predictable and act accordingly on all different types of browsers. No surprises, like out of place elements, inconsistent font heights, etc, when viewing the page on firefox, chrome, edge, safari, and even opera!

SEO optimization

Added Search Engine Optimization so search engines like google, duckduckgo, bing will better serve search results related to each specific field. Like logically cascading folder paths (ex. /lecture/01) and description meta tags to show what is actually relevant in search engine results.

Mobile friendly

52.2% of all internet traffic is through mobile. And students will surely browse the site through mobile to refer to the site when studying on the go, or when looking for resources. In this day and age, it's important to think mobile first.

Page tabulation

Different sections for the class has now been tabulated to better serve and display content to the user. By tabulating the classes, the users will know there are other class options by presenting it as an easy to discern option on the home page.

Printer friendly

Users can now print in ease, knowing that their printer will not waste space and paper. As this is an academic course website with lecture material, users may choose to print out the lecture notes to study and take notes, making printer friendly design a must have.

Error page handling

When someone lands on a page that is not reachable, not present, or unavailable the user will be presented with a useful 404 page. This page provides useful links to navigate to different pages, giving them more options than a standard 404 page.

Bootstrap integration

Used a modern and well established front end framework for styling. This way, future pages can be styled to fit the current designs, and can also be improved on as bootstrap updates. This will streamline development, allowing a uniform design.

Server side coding

Files paths are now handled using server side paths. This allows paths to be consistent over time, preventing confusing paths and folder traversals. This is great for future use cases. Error page redirection is also handled using server side coding (.htaccess).
